## 3. 具体清洗流程（按顺序，不可跳步）
## 步骤1：PDF 批量体检
命令：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\run_pdf_batch_check.ps1 -InputRoot D:\workspace\data
```

产物：

- `artifacts/pdf_validation_xxx/manual_review_sheet.csv`
- `artifacts/pdf_validation_xxx/validation_report.json`

## 步骤2：人工审核（强制）
打开 `manual_review_sheet.csv`，逐条处理：

1. `manual_need_ocr`：是否需要 OCR
2. `manual_body_start_page_ok`：正文起始页是否正确
3. `manual_skip_pages_ok`：封面/目录是否误删
4. `manual_review_status`：填 `done`
5. `manual_notes`：记录原因

标准文档：

- [pdf_manual_review_standard_v1.md](D:/workspace/lab-safe-assistant-github/docs/pdf_manual_review_standard_v1.md)

## 步骤3：回写特例规则
对反复出问题的 PDF，把规则写回：

- [pdf_special_rules.csv](D:/workspace/lab-safe-assistant-github/data_sources/pdf_special_rules.csv)

常用字段：

1. `force_ocr=true`
2. `body_start_page=<页码>`
3. `skip_pages=<页码列表>`

## 步骤4：导出统一知识库批次
命令：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\run_unified_batch.ps1 -DocumentInputRoot D:\workspace\data
```

产物：

- `artifacts/dify_kb_batch_xxx/knowledge_base_unified.csv`
- `artifacts/dify_kb_batch_xxx/run_report.json`

## 步骤5：提交清洗结果
提交到 Git 分支：

- `feat/clean-<姓名拼音>-<日期>`

至少提交这些文件：

1. 更新后的 `data_sources/pdf_special_rules.csv`
2. 本批次 `artifacts/.../run_report.json`
3. 人工审核结果 `manual_review_sheet.csv`（建议保留在批次目录）
4. 清洗说明（PR 描述中必须写）

## 4. 人工审核重点（只看关键）
你要优先看高风险文档：

1. 危化品
2. 电气
3. 气瓶
4. 应急处置
5. 辐射

判定原则：

1. 不可读就 OCR
2. 正文起始页错就修规则，不手改正文
3. 能规则化的不要反复人工改

## 5. 你的“提交合格标准”
必须同时满足：

1. `manual_review_status` 不能有未完成
2. 乱码高风险比例 <= 5%
3. 元数据完整率 >= 98%
4. 生成了可导入 CSV

## 6. 你交付后谁接手
你提交 PR 后，`发布与验收负责人` 会进行上线前验证和回归评测。
