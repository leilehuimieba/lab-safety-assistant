# V3 采集补链与人工补录说明

本文是给“信息收集员”看的快速说明，专门解释 `error/blocked` 条目该怎么处理。

## 1. 先看哪个文件
1. `data_sources/web_seed_urls_v3_prefetch_status.csv`：看哪些条目失败、被拦截。
2. `data_sources/web_seed_urls_v3_candidates.csv`：主候选池（流水线默认读取这个）。
3. `data_sources/web_seed_urls_v3_candidates_excel.csv`：Excel 友好版本（UTF-8 BOM）。

## 2. `error/blocked` 到底是什么意思
1. `error`：程序无法拿到有效正文（例如 412、反爬限制、网络策略）。
2. `blocked`：页面可访问，但内容受限（登录墙、权限墙、只返回壳页面）。

这两种都不是“数据无价值”，而是“当前自动抓取拿不到正文”。

## 3. 收集员应该怎么做（重点）
1. 保留原条目（不要直接删），用于追溯。
2. 找同主题的“可直达、可公开访问”的替代来源。
3. 在 `web_seed_urls_v3_candidates.csv` 里**新增一条**记录（新 `source_id`），不要覆盖原记录。
4. 在新增记录的 `tags` 里加 `替代WEBV3-xxx`，例如 `替代WEBV3-001`。
5. 优先替代来源顺序：`gov.cn > 部委官网 > 部委附件PDF > 高校转发件`。

## 4. 什么时候用“人工补录”
如果你能手动下载到 PDF/附件，但自动抓取拿不到正文，可以走人工补录：

1. 把 PDF 文件放到项目文档目录（例如 `data/manual_docs/`）。
2. 在 `data_sources/document_manifest.csv` 新增该文件元数据。
3. 清洗员用文档流水线解析，不走网页抓取。

这属于“换入库渠道”，不是放弃该来源。

## 5. 你问的“原编号还要不要”
要保留。

推荐做法：
1. 原编号保留（状态表里继续显示失败原因）。
2. 新增替代编号（例如 `WEBV3-019` 以后）。
3. 通过 `tags` 建关联：`替代WEBV3-001`。

## 6. 本期可直接执行的 collector 任务
见 `web_seed_urls_v3_prefetch_status.csv` 中 `recommended_assignee=collector` 的行。

当前优先：
1. `WEBV3-001`
2. `WEBV3-002`
3. `WEBV3-003`
4. `WEBV3-004`
5. `WEBV3-012`

## 7. 交付标准（最小可交付）
1. 至少补 3 条可访问替代来源。
2. 每条字段填完整（尤其 `priority`、`hazard_types`、`question_hint`）。
3. 提交 PR 时说明：替代了哪些旧编号、新增了哪些新编号。
