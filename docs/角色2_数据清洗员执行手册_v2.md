# 角色2：数据清洗员执行手册 v2

## 1. 你的职责（本阶段）
1. 对收集同学提交的来源做预抓取验证和状态归类。
2. 对可抓取来源执行统一清洗、切块、结构化入库。
3. 对被拦截/低质量来源组织人工补齐并回写状态。
4. 产出可导入 Dify 的批次 CSV，并提交 PR。

## 2. 你从哪里接单
固定输入文件：
- `https://github.com/leilehuimieba/lab-safety-assistant/blob/main/data_sources/web_seed_urls_v2_candidates.csv`
- `https://github.com/leilehuimieba/lab-safety-assistant/blob/main/data_sources/web_seed_urls_v2_prefetch_status.csv`
- `https://github.com/leilehuimieba/lab-safety-assistant/blob/main/data_sources/field_dictionary.json`

参考报告：
- `https://github.com/leilehuimieba/lab-safety-assistant/blob/main/docs/web_seed_v2_prefetch_report.md`

## 3. 标准执行流程（按顺序）
### 步骤1：拉取最新代码
```powershell
git checkout main
git pull origin main
```

### 步骤2：基础字段校验
```powershell
python scripts/validate_data_sources.py
```

目标：先拦住字段脏数据（类别、实验室类型、危险类型、语言等）。

### 步骤3：运行预抓取（可重复执行）
```powershell
python skills/web-content-fetcher/scripts/fetch_web_content.py `
  --url-file data_sources/web_seed_urls_v2_candidates.csv `
  --url-column url `
  --providers jina,scrapling,direct `
  --timeout 12 `
  --max-chars 30000 `
  --out-json artifacts/web_seed_v2_prefetch.json
```

说明：
- `status=ok`：可进入自动清洗流程。
- `status=blocked` 或 `requires_auth=true`：转人工补齐。
- `quality_score` 偏低：需要人工摘要和事实核验。

### 步骤4：更新状态分配表
更新文件：
- `data_sources/web_seed_urls_v2_prefetch_status.csv`

关键字段：
1. `status`
2. `quality_score`
3. `recommended_action`
4. `recommended_assignee`

建议分配规则：
1. `blocked`：`manual_capture_with_access`，交给收集员或人工访问同学。
2. `ok + 高质量`：`auto_clean_and_ingest_candidate`，进入自动入库。
3. `ok + 低质量`：`manual_summary_and_fact_check`，交给清洗+审核。

### 步骤5：网页清洗入库（Web Pipeline）
```powershell
python scripts/web_ingest_pipeline.py `
  --manifest data_sources/web_seed_urls_v2_candidates.csv `
  --output-dir artifacts/web_ingest_v2 `
  --fetcher-mode skill `
  --skill-script skills/web-content-fetcher/scripts/fetch_web_content.py `
  --skill-providers jina,scrapling,direct `
  --fetch-timeout 12 `
  --fetch-max-chars 30000
```

输出重点：
- `artifacts/web_ingest_v2/knowledge_base_web.csv`
- `artifacts/web_ingest_v2/run_report.json`

### 步骤6：统一合并批次（可选，建议每周一次）
```powershell
python scripts/unified_kb_pipeline.py `
  --web-manifest data_sources/web_seed_urls_v2_candidates.csv `
  --output-dir artifacts/unified_ingest_v2 `
  --web-fetcher-mode auto `
  --web-skill-script skills/web-content-fetcher/scripts/fetch_web_content.py `
  --web-skill-providers jina,scrapling,direct `
  --web-fetch-timeout 12 `
  --web-fetch-max-chars 30000
```

目标输出：
- `artifacts/unified_ingest_v2/knowledge_base_unified.csv`

## 4. 人工审核必须关注什么
优先审核高风险条目：
1. `risk_level >= 4`
2. 危化品、生物安全、气瓶、辐射、应急处置相关内容
3. 法规条款类页面（防止摘要误解条文）

审核标准：
1. 事实可追溯到来源链接。
2. 关键数字、条件、禁忌项不缺失。
3. 出现登录墙/权限墙时，不用猜测填充内容。

## 5. 交付物要求（提交 PR 时）
必须提交：
1. 更新后的 `data_sources/web_seed_urls_v2_prefetch_status.csv`
2. 清洗执行说明（写在 PR 描述）
3. 关键指标（写在 PR 描述）

PR 描述必须包含这些指标：
1. 总链接数
2. `ok/blocked/error` 数量
3. 可抓取率/被拦截率/低质量率
4. 本批次进入自动入库的条目数
5. 人工补齐清单（`source_id`）

说明：
- `artifacts/` 默认不进 Git（已忽略），请在 PR 描述粘贴关键结果。

## 6. 常见问题处理
1. `jina` 超时：脚本已支持 fallback，重跑即可。
2. Windows 编码报错：脚本已兼容控制台编码，优先使用仓库最新版。
3. 同一 URL 内容不稳定：以权威来源页面为准，必要时记录抓取时间并人工复核。

## 7. 交付给谁
清洗 PR 合并后，交给发布与验收负责人执行回归与发布门禁检查。
