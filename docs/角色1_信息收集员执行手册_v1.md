# 角色1：信息收集员执行手册 v1

## 1. 你要完成什么
你的任务只有三件事：

1. 找到高价值、可追溯的实验室安全资料
2. 按模板填写元数据
3. 提交到 GitHub 指定位置，等待清洗同学接力

你不负责清洗，不负责模型调参。

## 3. 要收集哪些信息（必须按优先级）
优先级：`P0 > P1 > P2`。每周至少 70% 来自 P0/P1。

P0（必须）：

- 校内实验室安全制度（准入、危化品、应急）
- 校内 SOP（通风柜、危化品领用、废液处理、电气安全）
- 常用 SDS/MSDS（乙醇、丙酮、盐酸、硫酸、硝酸、氢氧化钠等）

P1（强烈建议）：

- 国家/行业标准中与实验室安全直接相关条款
- 消防、电气、气瓶、辐射等规范
- 高校公开实验室安全培训资料

P2（可选）：

- 事故案例
- 设备说明书安全章节

## 4. 每条资料的“收集合格标准”
同时满足才算合格：

1. 来源可追溯（机构名 + URL）
2. 与实验室安全直接相关
3. 可用于问答（有规则、步骤、禁忌、应急等内容）
4. 版本或日期可识别

直接拒收：

1. 来源不明转载
2. 广告文/营销文
3. 与实验室安全弱相关内容

你提交两类内容：

1. 元数据 CSV（必须）
2. 资料下载链接/来源链接（必须）

## 6. 提交格式（必须）
分支命名：

- `feat/collect-<姓名拼音>-<日期>`

提交信息示例：

- `data: add 12 P0 docs and metadata (2026-03-17)`

PR 标题模板：

- `[数据收集] <日期> <姓名> 新增 <N> 条资料`

PR 描述必须包含：

1. 本次新增资料数量
2. P0/P1/P2 各占比
3. 是否包含版权敏感资料（如有，需注明仅保留引用链接）
4. 下游是否可直接清洗（yes/no）

## 7. 每天最小交付
每日最小合格线：

1. 新增 5 条有效来源
2. 每条来源都在 CSV 有记录
3. 每条都有 `question_hint`
