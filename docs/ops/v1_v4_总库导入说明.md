# V1~V4 总库导入说明（可直接执行）

## 1. 一键生成总库

在仓库根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\run_v1234_import_bundle.ps1
```

执行内容：

1. 重抓 `V1/V2/V3` 网页种子
2. 自动生成 `V1/V2/V3 prefetch_status.csv`
3. 合并 `curated + V1 + V2 + V3 + V4 pass`，按优先级去重
4. 输出可导入总库

## 2. 关键输出文件

- 总库 CSV：`artifacts/import_bundle_v1234/knowledge_base_import_ready.csv`
- 构建报告：`artifacts/import_bundle_v1234/import_bundle_report.md`
- 抓取状态：
  - `data_sources/web_seed_urls_v1_prefetch_status.csv`
  - `data_sources/web_seed_urls_v2_prefetch_status.csv`
  - `data_sources/web_seed_urls_v3_prefetch_status.csv`

## 3. 导入数据库（Dify）

1. 打开 Dify 控制台 -> 知识库
2. 新建或选择目标知识库（建议：`lab-safety-assistant-master-kb`）
3. 选择导入文件：`knowledge_base_import_ready.csv`
4. 分段策略建议：
   - 分段长度：800~1200
   - 重叠：80~120
   - 检索：混合检索（关键词 + 向量）
5. 导入完成后，抽测 10 个问题，确认能命中来源并返回可执行建议

## 4. 当前结果快照（2026-03-27）

- 总条目：`267`

- 质量门禁：通过

## 5. 注意事项

1. 部分政务站点会触发 `412/blocked`，会体现在 prefetch_status 中。  
2. 抓到的网页若出现乱码或壳内容，不建议直接覆盖高质量历史条目。  
3. 入库时优先使用总库 CSV，不直接导入原始 `fetch_results.jsonl`。
