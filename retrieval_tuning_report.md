## v2：高质量模式（混合检索 + bge-m3）2026-03-16

### 升级背景
v1 限制：OpenAI 兼容代理未提供 Embedding 模型，无法启用向量检索。
解决方案：通过 Ollama 本地部署 bge-m3（566M 参数，F16 量化，~1.2GB），配置为 Dify 的 Text Embedding 供应商。

### 配置过程
1. **Ollama 插件安装**：通过浏览器 JS fetch 下载 `.difypkg` 文件，绕过 Docker 网络隔离上传安装（`plugin_daemon` 容器无法访问 marketplace.dify.ai）
2. **bge-m3 模型下载**：`ollama pull bge-m3`，大小 1,157,672,605 bytes（F16 GGUF）
3. **Dify 模型供应商配置**：Ollama → 添加 Text Embedding 模型，Base URL = `http://host.docker.internal:11434`，Context Size = 4096
4. **知识库设置变更**：索引模式从经济升级至高质量，检索模式改为混合检索
5. **知识库 CSV 重新上传**：原文件为 2026-03-11 旧版（43 条），重新上传 2026-03-16 最新版（81 条，174,225 bytes），自动分段得 265 个 chunk

### 当前设置
- 索引模式: 高质量（High Quality）
- Embedding 模型: Ollama / bge-m3（本地）
- 检索模式: 混合检索（向量 0.7 + 关键词 0.3）
- Top K: 5
- Score 阈值: 0.5（启用）
- Rerank: 关闭（无 Rerank 模型）

### 关键发现与注意事项
- 自动分段（automatic mode）产生 265 个 chunk，bge-m3 嵌入全部完成，无错误
- 首次尝试（custom 分段，1156 chunks）触发 Ollama 500 错误，改用 automatic mode 后稳定
- bge-m3 加载时间约 8 秒（首次），后续嵌入请求延迟 < 1s/条
- Score 阈值 0.5 有效过滤低质量匹配，建议后续根据更多评测结果微调（0.45~0.55 区间）
